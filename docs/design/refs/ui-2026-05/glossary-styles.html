Trace 中的单个执行片段，可代表一次 LLM 调用、一次 Tool Call，或一次子 Agent 派生。Span 嵌套形成调用树。

点击名词当场展开一行解释卡。不打断布局、不弹层、不需要悬停。最适合做"渐进式学习"。

若实验版本的 触发集命中率 触发集命中率：「应该触发本 Skill 的样本中，确实被触发的比例」。数值越高越好。它与「误触发率」一起决定 Skill 的路由质量——单看命中率会掩盖误伤问题。 高于 80%，可推进到上线评估。

点击名词从右侧推出一张完整文档卡：定义、公式、示例、关联名词。重武器，专治"我真的要搞懂这个概念"的用户。

Skill 进入 SKILL 生命周期 的第二阶段时，需要先跑齐 评估覆盖度 ，再判断是否进入 Skill 优化器 。
当一次执行的 链路状态 标记为 CHAIN_ERROR · L13 时，问题节点已被定位。
适合用在
  • 核心概念（Skill / Trace / Evaluator）
  • 解释里要塞图表、代码、外链的场景
  • 用户文档 + 产品页打通时

  • 建议只对 10–20 个核心名词使用

边注 · Margin Annotation
把名词解释放到正文右侧，像古籍批注。视线不离开页面，所有名词解释一次性可见。适合静态长文档/说明页。
一览无遗 静态 占空间
// 实际效果 — 右侧批注与正文一一对应，可悬停高亮
系统检测到一次 CHAIN_ERROR， 失败的 Span 位于 Fuxi-Sub 子 Agent 的 background_output 工具调用上。建议运行 SMART RUN 一次性补齐缺失的评估维度。
CHAIN_ERROR 一次 Trace 的最终状态标记，表示链路中某节点出错且未被恢复。后缀 L13 = 第 13 个节点。
Span Trace 中的单个执行片段——一次 LLM 调用、一次 Tool 调用，或一次子 Agent 派生。
background_output 异步任务取结果的工具调用。超过 timeout 仍未完成会触发 Timeout 类故障。
SMART RUN 一键调度所有未完成评估维度（A/B、用例、触发、静态）的批量执行入口。
适合用在
  • "术语手册""新人 onboarding"等长文档
  • 智能诊断详情页（左 Trace 右注释）
  • 打印/导出 PDF 友好
注意
  • 需要 ≥ 1024px 宽，否则要降级
  • 批注数量 > 8 个时会显得拥挤
